What It Is

A new Account Usage view providing complete visibility into Cortex AI Guardrails activity, credit consumption, and prompt injection detection signals.

Key Insight: Each row represents a single guardrail scan for one tool use within an agent request. A request with multiple tool results produces multiple rows.
  • ■Per-scan granularity — Track every guardrail invocation individually
  • ■Token & credit breakdown — Input, output, cache read/write metrics
  • ■Prompt injection flags — Boolean signal when injection is detected
  • ■Role-level audit — Full metadata on who triggered the scan

How It Works
Architecture: Guardrails scan tool-use outputs for prompt injection signals, then send those signals to the LLM for adjudication.
flowchart LR
    A["Agent Request"] --> B["Tool Execution"]
    B --> C["Guardrail Scan"]
    C --> D{"Injection Signal?"}
    D -->|Yes| E["Flag + Log"]
    D -->|No| F["Pass + Log"]
    E --> G["USAGE_HISTORY View"]
    F --> G

Quick Start Queries

Recent guardrail scans (last 72 hours)

SELECT * FROM SNOWFLAKE.ACCOUNT_USAGE.CORTEX_AI_GUARDRAILS_USAGE_HISTORY WHERE USAGE_TIME >= DATEADD('hour', -72, CURRENT_TIMESTAMP()) ORDER BY USAGE_TIME DESC LIMIT 100;

Flagged prompt injection attempts

SELECT USER_NAME, AGENTIC_SOURCE, USAGE_TIME, GUARDRAIL_RESULTS FROM SNOWFLAKE.ACCOUNT_USAGE.CORTEX_AI_GUARDRAILS_USAGE_HISTORY WHERE GUARDRAILS_SIGNAL = TRUE AND USAGE_TIME >= DATEADD('hour', -72, CURRENT_TIMESTAMP()) ORDER BY USAGE_TIME DESC;

Credit usage by agentic source

SELECT AGENTIC_SOURCE, COUNT(*) AS scan_count, SUM(TOKEN_CREDITS) AS total_credits, SUM(TOKENS) AS total_tokens FROM SNOWFLAKE.ACCOUNT_USAGE.CORTEX_AI_GUARDRAILS_USAGE_HISTORY WHERE USAGE_TIME >= DATEADD('day', -7, CURRENT_TIMESTAMP()) GROUP BY AGENTIC_SOURCE ORDER BY total_credits DESC;

Best Practices

Do

  • ✓Monitor GUARDRAILS_SIGNAL daily for injection attempts
  • ✓Track credit consumption by source to manage costs
  • ✓Correlate REQUEST_ID with agent logs for full tracing
  • ✓Set up alerts on high-frequency flagged requests

Avoid

  • ✗Ignoring flagged signals without investigation
  • ✗Assuming one row = one request (multiple tool uses)
  • ✗Overlooking PARENT_REQUEST_ID for nested agents
  • ✗Running broad queries without time filters
